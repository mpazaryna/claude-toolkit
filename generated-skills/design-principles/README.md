# design-principles

Universal design theory for building intentional, cohesive interfaces across any platform.

## What This Skill Does

Provides foundational design principles that apply regardless of implementation platform (web, iOS, Android, desktop). This is the **theory layer** of the design-* skill family.

## When to Use

- Starting a new project and need design direction
- Making decisions about typography, color, spacing
- Reviewing designs for consistency and accessibility
- Establishing design systems or component libraries
- Training teams on design fundamentals

## Coverage

| Area | What You Get |
|------|--------------|
| **Typography** | Font selection by context, hierarchy, pairing rules |
| **Color** | Palette structure, dark mode, accessibility |
| **Hierarchy** | Visual weight, spacing, depth systems |
| **Motion** | Animation purpose, timing, easing |
| **Accessibility** | WCAG compliance, screen readers, keyboard nav |

## Related Skills

This skill is part of the `design-*` family:

- **design-principles** (this skill) - Universal theory
- **design-web** - Web/CSS implementation
- **design-swiftui** - SwiftUI/iOS implementation
- **design-review** - Audits and submission prep

## Quick Example

```
"Help me choose typography for a developer tool"
→ Routes to typography.md
→ Recommends IBM Plex Sans/Mono or JetBrains Mono
→ Explains why technical tools need precision and readability

"Set up a color system for my SaaS app"
→ Routes to color.md
→ Provides primary/accent/neutral structure
→ Includes dark mode guidance
```

## Philosophy

Design isn't decoration—it's communication. These principles help you make intentional choices that serve users rather than arbitrary aesthetic decisions.
