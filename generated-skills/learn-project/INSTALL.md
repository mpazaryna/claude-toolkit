## Customization

### Add New Technology Template

Create a new template for your tech stack:

```bash
cat > ~/.claude/skills/learn-project/assets/my-framework.md << 'EOF'
# My Framework Analysis Template

## Detection
Look for: my-framework.config.js

## Analysis Sections
- Configuration
- Architecture patterns
- Best practices
- Recommendations
EOF
```

Update `SKILL.md` detection logic:
```yaml
- **My Framework**: `my-framework.config.js` exists
```

And template integration:
```yaml
- My Framework → `/assets/my-framework.md`
```

## Troubleshooting

### Skill Not Recognized

**Solutions:**
1. Verify: `ls ~/.claude/skills/learn-project/SKILL.md`
2. Check YAML frontmatter is valid
3. Try explicit: "Use learn-project skill to analyze..."
4. Restart Claude Code

### Templates Not Loading

**Solutions:**
1. Verify: `ls ~/.claude/skills/learn-project/assets/`
2. Check all template files present:
   ```bash
   ls ~/.claude/skills/learn-project/assets/*.md
   ```
3. Ensure read permissions: `chmod +r ~/.claude/skills/learn-project/assets/*`

### Wrong Technology Detected

**Solutions:**
1. Check detection logic in SKILL.md
2. Manually specify: "Analyze as TypeScript project"
3. Update detection patterns if needed

### Analysis Incomplete

**Solutions:**
1. Ensure Claude has read access to all project files
2. Check for permission issues: `ls -la /path/to/project`
3. Provide absolute path instead of relative
