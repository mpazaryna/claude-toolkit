# design-review

Design audits, accessibility checks, and App Store submission preparation.

## What This Skill Does

Provides systematic review checklists for polishing designs before launch. This is the **quality assurance layer** of the design-* skill family.

## When to Use

- Preparing an app for App Store submission
- Running accessibility audits
- Final design quality review before launch
- Checking WCAG compliance

## Coverage

| Area | What You Get |
|------|--------------|
| **App Store** | Review Guidelines compliance, metadata checklist |
| **Accessibility** | WCAG audit, screen reader testing, contrast checks |
| **Design Quality** | Visual consistency, component review, state coverage |

## Related Skills

This skill is part of the `design-*` family:

- **design-principles** - Universal theory
- **web-design** - Web implementation
- **swift-ui** - SwiftUI implementation
- **design-review** (this skill) - Audits and submission

## Quick Example

```
"Review my app for App Store submission"
→ Routes to app-store.md
→ Provides compliance checklist
→ Covers common rejection reasons

"Run an accessibility audit"
→ Routes to accessibility-audit.md
→ WCAG 2.1 AA checklist
→ Screen reader testing guide
```

## Philosophy

Shipping quality requires systematic review. Checklists catch what tired eyes miss.
