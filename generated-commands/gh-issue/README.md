# GitHub Issue Command

> Fetch and display GitHub issue details in structured format for coding work

## Quick Start

```
/git:is 7
/git:is 123
```

## Overview

The GitHub Issue command fetches full details of GitHub issues and presents them in a structured, implementation-ready format optimized for coding agents to work with.

## What This Command Does

- Fetches complete issue details from GitHub
- Formats information for easy implementation
- Extracts key requirements and acceptance criteria
- Identifies affected files and components
- Highlights technical specifications
- Prepares context for coding work

## Output Format

- **Status** - Issue state, labels, assignees, milestone
- **Description** - Full issue body preserved
- **Key Requirements** - Extracted and bullet-pointed
- **Technical Context** - Code snippets, file paths, API details
- **Implementation Notes** - What needs to be done

## When to Use

Use this command when:
- Starting work on a GitHub issue
- Need context for issue implementation
- Want structured view of requirements
- Preparing to code a solution
