# Rebuild README Command

> Generate a developer-focused README.md for your project or module

## Quick Start

```
/rebuild-readme
/rebuild-readme /path/to/project
```

## Overview

The Rebuild README command analyzes your project directory and creates/updates a professional README.md with getting started instructions, architecture overview, development guide, and deployment information.

## What This Command Does

- Analyzes project structure and technologies
- Generates professional README.md
- Includes getting started instructions
- Documents architecture and components
- Lists available scripts and commands
- Provides deployment guidance
- Adds troubleshooting section
- Creates onboarding documentation

## README.md vs CONTEXT.md

| README.md | CONTEXT.md |
|-----------|------------|
| **Stable** - Changes rarely | **Dynamic** - Changes frequently |
| Project setup and architecture | Active work and recent changes |
| Getting started guide | Current bugs and blockers |
| General documentation | Testing focus areas |
| Deployment overview | Environment-specific notes |

## Generated Sections

1. **Project Overview** - Description, technologies, license
2. **Getting Started** - Prerequisites, installation, quick start
3. **Architecture** - High-level overview and components
4. **Development** - Scripts, testing, linting, build
5. **Deployment** - Environments, CI/CD, production notes
6. **API Documentation** - Endpoints, components, or CLI commands
7. **Contributing** - Guidelines and PR process
8. **Troubleshooting** - Common issues and solutions
