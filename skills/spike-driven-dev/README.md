# Spike-Driven Development Skill

## How to Use

Upload the `spike-driven-dev.skill` file to Claude's skills library:
1. Go to Skills settings in Claude
2. Click "+" to add a new skill
3. Upload `spike-driven-dev.skill`

## What's Inside

This skill contains:

### SKILL.md (Main)
- The Railroad Spike Principle
- 5-step spike workflow
- Success criteria checklist
- Common pitfalls

### Reference Files (Loaded as needed)
- **decision-criteria.md** - When to create spike tickets
- **spike-patterns.md** - Good vs bad spike characteristics, TDD flow
- **examples.md** - Real examples from your projects (Exercise Library, AttributeGraph crisis, MLX bug)

## Preview

The `spike-driven-dev-preview/` directory contains the extracted skill files if you want to read them before uploading.

## When Claude Will Use This

Claude will automatically use this skill when:
- Starting new features with unfamiliar patterns
- Testing architecture assumptions
- Integrating unfamiliar APIs/data sources
- Validating technical feasibility before full implementation
